# agent-qq v2.0.1 升级与测试报告

**日期：** 2026-06-14 | **版本跨度：** v1.x → v2.0.0 → v2.0.1

---

## 第一部分：升级报告

### 一、升级动机

v1.x 采用"AI 兜底"混合架构：未知消息全部透传给 Claude Code，存在以下问题：

1. **Token 浪费**：任意 QQ 消息都消耗 AI Token
2. **不可控**：无法预知哪些消息会触发 AI
3. **无任务管理**：缺少计划、状态追踪、异常处理
4. **/status Bug**：任务执行期间查询不到状态

### 二、架构变更

| 维度 | v1.x | v2.0 | v2.0.1 |
|------|------|------|--------|
| 路由模式 | if-elif + AI 兜底 | 指令字典 + 零 Token 拦截 | 同 v2.0 |
| AI 入口 | 3 个（/ask /code + 兜底） | 3 个（/ask /code /plan） | **1 个（仅 /plan）** |
| 未知消息 | 全部传给 Claude | 返回 /help 提示 | 同 v2.0 |
| 命令数量 | 7 条 | 16 条 | 14 条 |
| 任务持久化 | 纯内存 | 内存 + JSON 双写 | 同 v2.0 |
| 异常处理 | 无 | 三路检测 + 熔断 + 回滚 | 同 v2.0 |
| 状态监控 | 无 | 后台 5s 巡检 | 同 v2.0 |
| 日志管理 | 无 | 条数限制 + 终态清理 | 同 v2.0 |

### 三、v2.0.1 关键修正

v2.0.0 发布后发现 `/ask` 和 `/code` 仍可直接调用 AI，违反「仅 /plan 可调 AI」原则。v2.0.1 彻底移除这两个命令。

**AI 调用审计（command_router.py 全部 self._claude 调用）：**

| 方法 | 所在命令 | 是否 AI？ |
|------|---------|-----------|
| `.status()` | /status /token | ❌ `claude --version` |
| `._is_allowed_shell()` | /shell | ❌ 本地白名单 |
| `.shell()` | /shell | ❌ `create_subprocess_shell` |
| **`.ask()`** | **/plan /plan-start** | ✅ 唯一 AI 入口 |

> **结论：仅 /plan [自然语言] 能触发 AI。**
